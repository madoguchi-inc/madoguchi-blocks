# ブロック設計の共通ルール（色・フォント・アイコンは基本カスタマイズ可能に）

ブロックを新規追加・改修するときは、**色・フォント・アイコンを編集画面から変更できる**ことを基本方針とする。
設定しなくてもそのまま使える既定値を持たせつつ、記事・メディアごとに調整できる余地を必ず用意する。
このルールは、ブロックの edit.js / render.php / block.json / SCSS に触れる作業では**必ず遵守する**。

## 1. 色（Color）

- **アクセントカラー**: 主要色は CSS カスタムプロパティ `--md-brand` を参照する（ハードコード禁止）。
  - サイト全体の既定は「設定 > Madoguchi Blocks」のブランドカラー。
  - 各ブロックはサイドバー「カラー設定 > アクセントカラー」で上書きでき、未設定はブランドカラーへフォールバック。
- **背景色**: カード等は「カード背景色」をサイドバーで個別指定できるようにする（未設定はパターン既定／白）。
- 実装:
  - アクセントカラー等はルート要素の `style="--md-brand: …"` で供給し、SCSS は `var(--md-brand, <既定>)` で受ける。
  - 背景色などブロック単位で効かせたい値は**インライン style** で出す。
  - 色値は必ず `sanitize_hex_color()`（PHP）でサニタイズする。
  - インライン style で出す色は `tools/build-rest-css.js` の `INLINE_GUARDS` に登録する（REST 隔離ルール参照）。

## 2. フォント（Font）

- **文字サイズ**は原則としてサイドバーで変更できるようにする（`RangeControl`）。
  - 既存の実装パターンに合わせ、CSS カスタムプロパティ経由で供給する
    （例: 比較テーブル `--md-table-size`、口コミ本文 `--md-review-body`、CTAボタンは `font-size` をインライン）。
  - px 値で供給し、SCSS 側は `var(--md-… , <既定>)` で受ける。REST 用CSSは rem→px 固定化されるため齟齬が出ない。
- フォントファミリーは配信先テーマに依存させない（REST 用CSSがブロック内に基礎フォントを自前定義する）。
  独自にファミリーを固定したい場合はブロックのスタイルで明示し、REST 隔離の確認を行う。

## 3. アイコン（Icon）

- ブロックがアイコンを持つ場合は、**内蔵アイコンから選択**できるようにする（`SelectControl`）。
  さらに可能なら**画像アップロードでの差し替え**（`MediaUpload`）も用意する（例: 条件カード・おすすめリンクカード）。
- 内蔵アイコンの実体は `src/condition-card/icons.js`（`CardIcon` / `ICON_OPTIONS` / `PATHS`）に集約する。
  新しいアイコンはここに追加し、`ICON_OPTIONS` に選択肢も足す。全て stroke ベース・`currentColor` 追従で統一する。
- **サーバー描画（render.php）のブロックは、同じ図形の SVG を PHP 側にも持つ**ため、
  アイコンを追加・変更したら **`icons.js` と該当 render.php の両方を必ず同期**する
  （例: `blocks/recommend-card/render.php` の `$icon_paths`）。片方だけの更新は禁止。
- アイコン色は `currentColor` とし、`--md-brand` で着色できるようにする（ハードコードしない）。

## 4. テキスト入力（キャンバス上でのインライン編集）

- ブロックの表示テキスト（見出し・タイトル・本文・リード・ラベル等）は、
  **エディタのキャンバス上で直接編集できる**ように `RichText` で実装する。
  サイドバーの `TextControl` だけで入力させる設計にしない（URL・計測ID等の非表示メタ値は `TextControl` で可）。
- **インライン書式・インライン要素を許容する**。本文・説明・リードなど文章系は `allowedFormats` を
  過度に絞らず、太字・リンク等のインライン装飾を使えるようにする（`allowedFormats={[]}` で全面禁止にしない）。
  - 短いラベルや、構造化データ／属性にそのまま使う値だけは、必要に応じて `allowedFormats` を絞ってよい。
- render.php（動的ブロック）側の出力方針:
  - **表示用テキスト**はインライン書式を活かすため `wp_kses()` に安全な許可リストを与えて通す
    （例: `<br>`, `<strong>`, `<em>`, `<a href|target|rel>` など）。`wp_strip_all_tags()` で一律に潰さない。
  - **属性値・JSON-LD・aria 等に使う値**はプレーン化する（`wp_strip_all_tags()`）。表示とデータで使い分ける。
- save 済み（静的）ブロックは `RichText.Content` で保存し、エディタと同じマークアップにする。

## チェックリスト（ブロック新規・改修時）

- [ ] 主要色は `--md-brand` 参照。アクセントカラーをサイドバーで上書き可能にしたか
- [ ] 背景色などブロック単位の色をインライン style で出し、`INLINE_GUARDS` に登録したか
- [ ] 文字サイズを `RangeControl`＋CSS変数で変更可能にしたか
- [ ] アイコンは内蔵から選択可（必要なら画像差し替えも）にしたか
- [ ] アイコンを足したら `icons.js` と render.php の SVG を同期したか
- [ ] 表示テキストはキャンバス上の `RichText` で編集でき、文章系はインライン書式を許容したか
- [ ] 表示は `wp_kses`（許可リスト）／データ用途は `wp_strip_all_tags` と使い分けたか
- [ ] 色値・URLをサニタイズ（`sanitize_hex_color` / `esc_url_raw`）したか

## 注意

- 「変更できる」ことが目的であって、既定で崩れないことが前提。未設定時に成立する既定値を必ず持たせる。
- 色・フォント・アイコンの変更を追加したら、REST 隔離ルール（`.claude/rules/rest-isolation.md`）の確認も行う。
